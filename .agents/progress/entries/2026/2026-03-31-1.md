本轮把 `v1.0.0` 从文档计划推进到了最小实现闭环, 重点包括:

- 给 `profile.yaml` 引入 `project_version`, 并固定机器真源格式为 `major.minor.patch`
- 给 `init` 增加 `--project-type embedded`, 可以直接生成嵌入式治理骨架
- 新增嵌入式 scaffold 文档:
  - `docs/DEVELOPMENT_PLAN.md`
  - `docs/PROTOCOL_SPEC.md`
  - `docs/HARDWARE_BRINGUP.md`
  - `docs/VALIDATION_PLAN.md`
  - `docs/schema/protocol.schema.json`
- 扩展 `validate`, 增加对 embedded 项目的最小检查:
  - profile 必填字段
  - 必需文档存在性
  - `project_version` 格式
  - 协议文档 / Schema / 验证计划版本一致性
  - `VALIDATION_PLAN` 中 `M0 / M1 / M2 / M3` 结构
- 让受管模板统一引用 `project_version`
- 补了目录导航、升级索引、贡献指南和嵌入式指南中的相关说明

这轮实现的边界也保持得比较明确:

- 做了 `P0` 主干能力和 `P1` 最小护栏中最关键的部分
- 没做自动版本 bump
- 没做提交消息自动生成器
- 没做 Git Hook 自动安装
- 没做根目录目录迁移
