Added a dedicated root document, `AI_QUICKSTART.md`, so the user can copy one stable prompt into a brand new IDE session.

This prompt tells the new AI how to:

1. detect whether the current project has already been bootstrapped;
2. run `vibe-governance bootstrap --type ... --target .` in the current project when needed;
3. read the generated local `START_HERE.md` first;
4. treat `.agents/` as the source of truth instead of editing generated adapter files directly.

Updated `README.md`, `QUICKSTART.md`, and `DIRECTORY_STRUCTURE.md` so people can clearly distinguish:

- `QUICKSTART.md` is for humans;
- `AI_QUICKSTART.md` is the reusable prompt for the AI inside the new IDE.
