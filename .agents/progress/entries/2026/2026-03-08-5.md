Corrected the primary usage model to match the real workflow:

1. keep Taotaotao as a local governance tool repository;
2. open a brand-new project directory in a new IDE;
3. run `vibe-governance bootstrap --type ... --target .` inside that directory;
4. let the new IDE's AI read the generated `START_HERE.md` first.

This avoids the earlier mismatch where the docs implied that project creation should mainly happen back inside the Taotaotao repository.

The generated project entry files now also record the local Taotaotao source path, so the new IDE's AI can read the tool repository itself when it needs deeper context about the generator.
