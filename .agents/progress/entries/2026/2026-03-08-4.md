Rewrote the root human-facing Markdown files so they answer the real first-use question:

How do I create a new project, open it in a new IDE, and get AI into the correct working state?

The updated docs now center on a single path:

1. run `bootstrap` in the new project directory;
2. open the generated project root in a fresh IDE;
3. tell AI to read `START_HERE.md`;
4. align on `.agents/` truth before coding.

This reduces the old mismatch where the root docs explained the governance repository itself more than the actual day-to-day usage path.
