Removed the duplicate legacy project-creation path from the CLI.

The repository now keeps a single project creation flow:

1. install `Taotaotao` as a local tool;
2. open a fresh project directory in a new IDE;
3. run `vibe-governance bootstrap --type ... --target .`.

This reduces conceptual overlap, simplifies the CLI surface, and keeps the code aligned with the actual intended workflow without removing the core capability to create a fully bootstrapped project.
