Added a `smoke` CLI command for the shortest end-to-end test path.

The command validates the current governance project, bootstraps a temporary sample project under `.tmp-tests/smoke/`, validates that generated project, and confirms its dry-run sync status is clean.

This keeps the test workflow simple enough for daily use while still checking the two most important guarantees:

1. the current repo is still valid;
2. a newly generated project can still boot and pass validation.
