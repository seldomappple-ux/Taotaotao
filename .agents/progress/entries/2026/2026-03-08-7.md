Updated the deep documentation under `docs/` so it matches the current workflow:

1. install `Taotaotao` once as a local tool;
2. open a fresh project directory in a new IDE;
3. run `vibe-governance bootstrap --type ... --target .`;
4. let the new project's AI read `START_HERE.md` first.

The main fixes were in `USAGE_GUIDE.md`, `EMBEDDED_GUIDE.md`, and `CODE_WALKTHROUGH.md`, where the older `init`-first explanation no longer matched the current product shape.
